# Алгоритм ротации блюд в расписании

## Обзор

Улучшенный алгоритм выбора рецептов для ежедневного расписания с рандомным выбором и умной ротацией.

## Основные принципы

### 1. Жесткое ограничение на повторение
- **Гарантия**: Одно и то же блюдо **никогда** не будет выбрано 2 дня подряд
- Если есть хотя бы один альтернативный рецепт, вчерашние рецепты исключаются из выбора
- Только если ВСЕ рецепты использовались вчера (крайне редкий случай), система выберет из всех доступных

### 2. Взвешенная рандомизация
Каждому рецепту присваивается вес в зависимости от того, когда он использовался последний раз:

| Последнее использование | Вес | Описание |
|------------------------|-----|----------|
| Никогда не использовался | 5.0 | Максимальный приоритет |
| 21+ дней назад | 3.0 | Очень высокий приоритет |
| 14-21 дней назад | 2.0 | Высокий приоритет |
| 7-14 дней назад | 1.5 | Повышенный приоритет |
| 5-7 дней назад | 1.0 | Нормальный приоритет |
| 3-5 дней назад | 0.8 | Средний приоритет |
| 3 дня назад | 0.5 | Низкий приоритет |
| 2 дня назад | 0.3 | Очень низкий приоритет |
| Вчера | 0.1 | Минимальный (фильтруется) |
| Сегодня | 0.05 | Почти невозможно |

### 3. Равномерная ротация
- Система отслеживает использование рецептов за последние **30 дней**
- Рецепты, которые давно не использовались, получают экспоненциально больший вес
- Это обеспечивает равномерное распределение всех блюд в меню

## Как это работает

### Шаг 1: Получение доступных рецептов
```
- Загружаются все активные рецепты для данного приема пищи
- Фильтруются по члену семьи (adult/baby/all)
- Учитывается связь many-to-many с meal_times
```

### Шаг 2: Анализ истории
```
- Получается история использования за 30 дней
- Определяются рецепты, использованные вчера
- Рассчитывается количество дней с последнего использования для каждого рецепта
```

### Шаг 3: Расчет весов
```
- Каждому рецепту присваивается вес по формуле выше
- Рецепты из вчерашнего дня помечаются
```

### Шаг 4: Фильтрация
```
- Если есть рецепты НЕ из вчерашнего дня → используются только они
- Если все рецепты из вчерашнего дня → используются все (с предупреждением в логе)
```

### Шаг 5: Рандомный выбор
```
- Выполняется взвешенный случайный выбор
- Вероятность выбора рецепта пропорциональна его весу
- Рецепты с большим весом имеют больше шансов быть выбранными
```

## Примеры

### Пример 1: Обычная ситуация
Доступно 10 рецептов для завтрака:
- 3 рецепта использовались 2-3 дня назад (вес 0.3-0.5)
- 1 рецепт использовался вчера (исключается)
- 4 рецепта использовались 7-14 дней назад (вес 1.5)
- 2 рецепта не использовались 20+ дней (вес 3.0)

**Результат**: Высокая вероятность выбора одного из 2 давно не использованных рецептов

### Пример 2: Мало рецептов
Доступно 3 рецепта:
- Рецепт A: использовался вчера (исключается)
- Рецепт B: использовался 3 дня назад (вес 0.5)
- Рецепт C: использовался 10 дней назад (вес 1.5)

**Результат**: Рецепт C имеет в 3 раза больше шансов быть выбранным, чем B

## Логирование

Система записывает в лог:
- Количество найденных рецептов
- Количество рецептов, использованных вчера
- Выбранный рецепт с его весом и днями с последнего использования
- Предупреждения, если все рецепты были использованы вчера

## Преимущества

✅ **Никогда не повторяется 2 дня подряд** (жесткое ограничение)  
✅ **Равномерная ротация** всех блюд в меню  
✅ **Рандомность** - меню не предсказуемо  
✅ **Адаптивность** - автоматически подстраивается под количество рецептов  
✅ **Прозрачность** - подробное логирование для отладки  

## Технические детали

### Функции

1. **`selectRecipeForMeal(mealTimeID, familyMember, currentDate)`**
   - Главная функция выбора рецепта
   - Принимает текущую дату для определения "вчера"

2. **`getRecentlyUsedRecipes(mealTimeID, lookbackDays)`**
   - Возвращает map[recipeID]daysSinceLastUse
   - Анализирует последние 30 дней

3. **`getYesterdayRecipes(currentDate, mealTimeID)`**
   - Возвращает set рецептов, использованных вчера
   - Используется для жесткой фильтрации

4. **`selectRecipeWithImprovedRotation(recipes, recentlyUsed, yesterdayRecipes)`**
   - Выполняет взвешенный рандомный выбор
   - Применяет все правила ротации

## Настройка

Параметры можно изменить в коде:

```go

recentlyUsed := s.getRecentlyUsedRecipes(mealTimeID, 30) // 30 дней

// Можно настроить под свои предпочтения
```
